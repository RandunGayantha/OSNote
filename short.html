Introduction to Operating Systems

What is an Operating System?

An OS is a software layer that acts as a bridge between computer hardware and user programs. It manages resources and provides services for applications.

OS Functions

  • Resource Manager: Allocates CPU, memory, I/O devices
  • Extended Machine: Provides easy-to-use interface
  • Controls program execution
  • Acts as intermediary between users and hardware

OS Objectives

  • Convenience: Makes computer easier to use
  • Efficiency: Maximizes resource utilization
  • Extensibility: Allows adding new features

OS Evolution

  1. 1st Gen (1945-55): Vacuum tubes, no OS
  2. 2nd Gen (1955-65): Transistors, batch systems
  3. 3rd Gen (1965-80): ICs, multiprogramming
  4. 4th Gen (1980-): Personal computers
  5. 5th Gen (1990-): Mobile computing

Operating System Zoo

Different types of OS for different purposes:

  • Mainframe OS: For large-scale processing
  • Server OS: Multi-user environments
  • PC OS: Windows, macOS for personal use
  • Smartphone OS: Android, iOS for mobile
  • Real-Time OS: For time-critical systems
  • Embedded OS: For small devices like smartwatches

Threads

User-Level Threads

  • Managed by user libraries
  • Fast and portable
  • Blocking affects entire process

Kernel-Level Threads

  • Managed by OS kernel
  • Independent scheduling
  • Higher overhead

Hybrid Threads

  • Combines both approaches
  • Flexible and efficient
  • Example: Thread libraries

Process Scheduling

Scheduling Types

  • Long-Term: Decides which jobs to admit
  • Medium-Term: Manages memory by suspending/resuming processes
  • Short-Term: Allocates CPU to ready processes

Scheduling Algorithms

First-Come-First-Served (FCFS)

  • Simple, executes in arrival order
  • Convoy effect - long jobs delay short ones
  • Non-preemptive

Shortest Job First (SJF)

  • Chooses shortest CPU burst first
  • Optimal for minimizing waiting time
  • Requires knowing burst times in advance

Priority Scheduling

  • Executes based on priority levels
  • May cause starvation of low-priority jobs
  • Aging can prevent starvation

Round Robin (RR)

  • Each process gets fixed time quantum
  • Balances responsiveness and overhead
  • Time quantum size affects performance

Real-Time Scheduling

  • Rate Monotonic: Shorter period = higher priority
  • Earliest Deadline First: Prioritizes nearest deadlines
  • Used in time-critical systems

Memory Allocation Techniques

Fixed Partitioning

  • Memory divided into fixed-size partitions
  • Simple but causes internal fragmentation
  • Degree of multiprogramming limited by partitions

Variable Partitioning

  • Memory allocated based on process needs
  • No internal fragmentation
  • Causes external fragmentation

Paging

  • Divides memory into fixed-size frames
  • Uses page tables for address translation
  • Eliminates external fragmentation

Virtual Memory

Extends RAM by using disk space, allowing programs to run even if not entirely in memory.

Page Replacement Algorithms

  • FIFO: Replaces oldest page (may suffer Belady's anomaly)
  • Optimal: Replaces page not used for longest time (requires future knowledge)
  • LRU: Replaces least recently used page
  • Clock: Gives pages a second chance before replacement

Thrashing

Occurs when excessive page swapping reduces CPU utilization. Can be avoided using:

  • Working Set Model
  • Page Fault Frequency algorithm
  • Proper memory allocation

File Systems

File Attributes

  • Name, Identifier, Type
  • Location, Size, Protection
  • Timestamps (created, modified, accessed)
  • Stored in File Control Block (FCB)

File Operations

Basic Operations

  • Create, Read, Write
  • Seek, Delete, Truncate

Advanced Operations

  • Lock/Unlock
  • Memory Mapping

File Allocation Methods

Contiguous

  • Files stored in continuous blocks
  • Fast but causes external fragmentation
  • Used in ISO9660 (CD-ROMs)

Linked

  • Files stored in linked blocks
  • No fragmentation but slow access
  • Used in FAT file system

Indexed

  • Uses index tables for file blocks
  • Efficient for large files
  • Used in NTFS, ext4

Directory Structures

  • Single-Level: All files in one folder
  • Two-Level: One folder per user
  • Hierarchical: Tree structure (most common)
  • DAG: Directed Acyclic Graph (supports links)

File System Types

  • FAT: Simple, uses linked allocation
  • NTFS: Advanced, uses indexed allocation with extents
  • ext4: Linux, uses inodes for metadata

Confidentiality

Integrity

Protects data from unauthorized alteration

Exam-Ready Questions & Answers

Q: What is the difference between a program and a process?

A: A program is a passive set of instructions stored on disk, while a process is an active instance of a program loaded in memory with its own resources and state.

Q: Explain the concept of virtual memory.

A: Virtual memory is a technique that uses disk space to extend RAM, allowing programs to run even when not entirely loaded in physical memory. It provides each process with its own address space and uses paging to map virtual addresses to physical ones.

Q: Compare FCFS and SJF scheduling algorithms.

A: FCFS (First-Come-First-Served) executes processes in arrival order, which is simple but can lead to long waiting times for short jobs behind long ones. SJF (Shortest Job First) prioritizes processes with the shortest CPU burst, minimizing average waiting time but requires knowing burst times in advance.

Q: What are the three main file allocation methods?

A: The three main methods are:

  1. Contiguous - files stored in continuous blocks (fast but causes fragmentation)
  2. Linked - files stored in non-contiguous blocks linked by pointers (flexible but slow access)
  3. Indexed - uses index tables to locate file blocks (efficient for large files)

Q: What is thrashing and how can it be prevented?

A: Thrashing occurs when excessive page swapping reduces CPU utilization. It happens when processes don't have enough memory frames. Prevention methods include:

  • Using the Working Set Model to allocate enough frames
  • Implementing Page Fault Frequency algorithm
  • Proper memory allocation policies
  • Increasing physical memory or reducing system load

Q: Explain the difference between internal and external fragmentation.

A: Internal fragmentation occurs when allocated memory blocks are larger than needed, wasting space within the allocation (common in fixed partitioning). External fragmentation happens when free memory is broken into small, non-contiguous pieces that can't be used for new allocations (common in variable partitioning).

Q: What are the components of the CIA triad in security?

A: The CIA triad consists of:

  • Confidentiality - ensuring data is only accessible to authorized users
  • Integrity - protecting data from unauthorized modification
  • Availability - ensuring systems and data are accessible when needed

Q: Compare user-level and kernel-level threads.

A: User-level threads are managed by user libraries without kernel involvement, making them fast and portable, but blocking one thread blocks the entire process. Kernel-level threads are managed by the OS, allowing independent scheduling and better concurrency, but with higher overhead due to system calls.
